Vibe-coding garbage analysis (`src/analyze_vibe.py` → `outputs/vibe_coding_analysis.md`):

| Tier | Count | % |
|---|---:|---:|
| garbage (score ≥5) | 18 | 1.8% |
| suspicious (3-4) | 132 | 13.2% |
| legitimate (0-2) | 849 | 85.0% |

Key insights worth raising in final report:

1. **Stars-bucket cross-cut**: garbage concentrates in the **1000-4999 stars range (9.9%)**, not at the very top (0% in ≥10k bucket) nor the long tail (1.0% in 100-499). Sweet spot for farming = "impressive enough to trend, low enough to escape scrutiny".
2. **54.4% of trending repos have ZERO topics.** Authors don't tag; this is a structural problem for any topic-based analysis.
3. **`agent` (29) + `claude` (25) + `codex` (16) + `skills/skill` (33)** dominate generic-name pattern, confirming 2026 AI-tooling hype cycle.
4. Eight "famous-nothing" repos (stars >1000 + empty description) include official-looking ones (`cursor/cookbook`, `deepseek-ai/awesome-deepseek-agent`) — rapid-shipping cultural pattern, not necessarily malicious.
5. False-positive concerns: `awesome-*` lists (11 hits) and academic research repos like `Jiawei-Yang/FD-Loss` (low forks = paper readers, not farming).

Data lives at: `data/processed/repos.csv` (999×19), `repo_languages.csv`, `repo_topics.csv`, `vibe_scores.csv`.
