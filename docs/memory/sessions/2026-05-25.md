# Session Log: 2026-05-25

## Summary

- Bug-fix session on `dashboard/app.py` — three user-reported issues that all traced to one root cause: tables had no working GitHub links.

## Bugfix: Dashboard GitHub Links Broken

**Symptoms reported (PC / Streamlit Cloud):**
1. "Display bug" in the explorer table — `repo` cell rendered as a broken/non-navigable link.
2. Sidebar filter widgets felt unresponsive (no visible link change in the result table).
3. Top 15 Vibe table and Famous-Nothing table had no clickable GitHub links at all.

**Root cause:**
- Explorer table used `st.column_config.LinkColumn` on the `full_name` column, but `full_name` is `"owner/repo"` — not a URL. Streamlit treated it as a relative href, producing 404s on PC.
- Top 15 Vibe and Famous-Nothing tables never had a link column; `vibe_scores.csv` only carries `full_name` (no `html_url`).

**Fix:**
- Explorer: switch `LinkColumn` to the real `html_url` column, keep `full_name` as a plain text "repo" column for readability.
- Top 15 Vibe + Famous-Nothing: merge `vibe` rows against `repos.set_index("full_name")["html_url"]` to attach the real URL, then add a `LinkColumn("GitHub", display_text="🔗 打開")` as the first column.
